Un backup NAND valido dovrebbe avere una delle dimensioni riportate di seguito.
{: .notice--primary}

Occasionalmente potrai ottenere un backup di dimensioni diverse da quelle qui elencate. Non è un problema se il backup è più grande del dovuto, ma non deve essere mai *più piccolo* delle dimensioni qui riportate.
{: .notice--info}

Tieni presente che un backup NAND creato con l'opzione *min size* avrà sempre la minor dimensione possibile per risparmiare spazio sulla scheda SD, e potrebbe non corrispondere alle dimensioni di altri backup della NAND della tua console. Assicurati comunque che abbia una dimensione corrispondente a una di quelle riportate di seguito.
{: .notice--info}

Il nome del marchio si riferisce al produttore del chip della NAND nella tua console, e **non** a quello della tua scheda SD.
{: .notice--warning}

**Il file di backup deve corrispondere ad una delle dimensioni riportate su questa pagina; se così non fosse, cancellalo e creane uno nuovo!**
{: .notice--danger}

## 2DS:    

    + NAND Toshiba:     
      988.807.168 bytes = 943 MiB    

    + NAND Toshiba:    
      1.979.711.488 bytes = 1.84 GiB    

    + NAND Samsung:    
      1.000.341.504 bytes = 954 MiB    

    + NAND Samsung:    
      1.300.234.240 bytes = 1.21 GiB
